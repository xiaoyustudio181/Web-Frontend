1. <!DOCTYPE> 不是HTML标签。它为浏览器提供一项声明，即 HTML 是用什么版本编写的。
    Web 世界中存在许多不同的文档，只有了解文档的类型，浏览器才能正确地显示文档。
    HTML5
    <!DOCTYPE html>
    HTML4.01
    <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd">
    XHTML1.0
    <!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">
  2. <head> 头部元素的容器。可包含脚本，指示浏览器在何处找样式表，提供元信息等。
    这些标签可以添加到 head 部分：<title>、<base>、<link>、<meta>、<script> 以及 <style>。
  3. <meta http-equiv="Content-Type" content="text/html; charset=utf-8">
    设定页面使用的字符集。
  4. <meta http-equiv="expires" content="Wed, 26 Feb 1997 08:21:57 GMT">
    设定网页缓存到期时间（必须使用 GMT 时间格式），过期就会直接到网站服务器重新下载页面内容，而不是从 Cache 调用。
    content 设为 -1 表示网页在任何时候都不能被 Cache 存储，和设置过期时间效果一样。content 设为正数表示多少秒后缓存过期。
  5. <meta http-equiv="Set-Cookie" content="cookievalue=abc;expires=Wednesday, 21-Oct-98 16:14:21 GMT; path=/">
    如果网页过期，存储的 cookie 将被删除。
  6. <meta meta http-equiv="pragma" content="no-cache">
    禁止浏览器从本地缓存中调阅页面内容，访问者将无法脱机浏览。
  7. <meta http-equiv="refresh" content="3;url=https://www.baidu.com">
    定时让网页自动链接到其它网页。
  8. <meta http-equiv="Window-target" content="_top">
    强制页面在当前窗口以独立页面显示，防止别人在框架里调用你的页面。
  9. <meta name="keywords" content="life, universe, mankind, plants">
    告诉搜索引擎的关键字。
  10. <meta name="description" content="This page is about the meaning of life, the universe, mankind and plants.">
    告诉搜索引擎的主要内容。 meta 标签最重要的就是 keywords 和 description 的设定。这两个语句可以让搜索引擎能准确的发现你，吸引更多的人访问你的站点! 根据现在流行搜索引擎(Lycos，AltaVista等)的工作原理，搜索引擎首先派出机器人自动在 www 上搜索，当发现新的网站时，便检索页面中的 keywords 和 decription，并将其加入到自己的数据库，然后再根据关键词的密度将网站排序。也就是说：1.如果你的页面中根本没有 keywords 和 description 的 meta 标签，那么机器人是无法将你的站点加入数据库，网友也就不可能搜索到你的站点。2.如果你的关键字选的不好，关键字的密度不高，被排列在几十甚至几百万个站点的后面被点击的可能性也是非常小的。所以，我们必须记住添加 keywords 和 description 的 meta 标签，并尽可能写好关键字和简介。写 keywords 的禁忌是：1.不要用常见词汇。例如 www, homepage, net, web 等；2.不要用形容词，副词。例如最好的，最大的等；3.不要用笼统的词汇，要尽量精确。例如不用"摩托罗拉手机"，改用"V998"等。寻找合适的关键词的一个技巧是：到 lycos, Alta, infoseek 等搜索引擎，搜索与你内容相仿的网站，查看排名前十位的网站的meta 关键字，copy下来用到自己的站点上。
  11. <meta name="robots" content="all">
    告诉搜索引擎哪些页面需要索引，哪些页面不需要索引。
    all(默认)
    文件将被检索，且页面上的链接可以被查询
    none
    文件将不被检索，且页面上的链接不可以被查询
    index
    文件将被检索
    follow
    页面上的链接可以被查询
    noindex
    文件将不被检索，但页面上的链接可以被查询
    nofollow
    文件可以被检索，但是页面上的链接不可以被查询
  12. <meta name="revisit-after" content="7 days">
    通知搜索引擎多少天访问一次。
  13. <meta name="author" content="xiaoyu,petezh@yeah.net">
    标注网页的作者。
  14. <meta name="viewport" content="width=device-width, initial-scale=1">
    name="viewport" 的 meta 标签面向移动设备浏览器，用于指定用户是否可以缩放 Web 页面，以及缩放的最大和最小比例，它还表示文档针对移动设备进行了优化，content 值是由指令及其值组成的以逗号分隔的列表。
    例：<meta name="viewport" content="width=240,height=320,user-scalable=yes,initial-scale=2.5,maximum-scale=5.0,minimun-scale=1.0">
    width 和 height
    指令分别指定视区的逻辑宽度和高度，其值要么是以像素为单位的数字，要么是一个特殊的标记符号。
    width=device-width
    指示视区宽度应为设备的屏幕宽度。
    height=device-height
    指示视区高度为设备的屏幕高度。
    user-scalable
    指定用户是否可以缩放视区，即缩放 Web 页面视图，值为 yes 时允许缩放，值为 no 时不允许缩放。
    initial-scale
    用于设置Web页面的初始缩放比例，默认的初始缩放比例值在不同手机浏览器上不同，通常情况下设备会在浏览器中呈现出整个 Web 页面，设为 1.0 则显示未经缩放的 Web 文档。
    maximum-scale 和 minimum-scale
    指令用于设置用户对Web页面缩放比例的限制，值范围在 0.25 至 10.0 之间，与 initial-scale 相同，这些指令的值是应用于视区内容的缩放比例。
  15. <title> 定义文档的标题。title 元素在所有 HTML/XHTML 文档中都是必需的。
    作用：一，定义浏览器工具栏中的标题；二，提供页面被添加到收藏夹时显示的标题；三，显示在搜索引擎结果中的页面标题。
  16. <html> 元素指从标签开始到结束的所有代码。
    HTML代码中的所有连续的空行(换行)也被显示为一个空格。
  17. HTML标签属性。
    提供了有关 HTML 元素的更多的信息。属性总是以名称/值对的形式出现，属性值应该始终被包括在引号内。双引号是最常用的，不过使用单引号也没有问题，比如：name="value"。属性总是在 HTML 元素的开始标签中规定。
